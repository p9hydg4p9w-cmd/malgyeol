## 6.1 추가 기능
- 새 표현을 사용자가 직접 등록
- 등록 직후 분석과 표현 사전에 반영
- 브라우저 localStorage에 보관
- 개별 삭제와 전체 삭제
- 추가 표현 목록을 JSON 파일로 내보내기

GitHub Pages는 서버가 없는 정적 사이트이므로 여러 사람의 제보가 운영자에게 자동 전송되지는 않습니다. 현재 버전은 각 기기에 저장하고 JSON으로 내보내도록 구성했습니다.


## 6.2 제보 검토 반영
- 제보 파일의 6개 표현을 뜻과 실제 사용 맥락에 맞춰 재분류
- `절다`는 일반 동사라 자동 탐지어에서 제외
- `찐따`, `멍청이`는 개인 모욕으로 분류
- `바보`, `미친`은 문맥 확인으로 분류
- `창녀`는 성적 낙인과 여성 비하 표현으로 설명 보강
- 관련 표현을 추가하거나 기존 설명을 수정
- 분석 카드에서 `뜻`, `왜 바꾸나요`, `추천`을 분리해 표시
- 새 표현 추가 양식에 `뜻·사용 맥락` 입력란 추가
- 과거 브라우저에 저장된 잘못된 6개 제보는 자동 제거 후 검토된 기본 DB를 사용
